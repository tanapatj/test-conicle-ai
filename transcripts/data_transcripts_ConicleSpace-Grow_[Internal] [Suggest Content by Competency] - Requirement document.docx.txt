Objective 


ต้องการเพิ่มเติมให้พนักงานสามารถค้นหาเนื้อหาที่เกี่ยงข้องกับการพัฒนาความสามารถ (Competency) ตามระดับที่ตนเองยังขาดได้ นอกเหนือจากการเลือกดูเนื้อหาจากส่วนการสร้าง Action ใน Development Plan
Statement of work
In Scope (Functional)
เพิ่มเติมให้พนักงานทั่วไป (User) สามารถได้รับการแนะนำเนื้อหาที่เกี่ยวข้องกับข้อมูลความสามารถตามระดับที่ตนเองยังขาดได้อยู่ โดยจะเพิ่มเติมข้อมูลในส่วน All Competencies ให้สามารถกดดูรายละเอียดเนื้อหาที่แนะนำเพิ่มเติมได้

Note : แตกต่างจากการแนะนำเนื้อหาใน Feature : Development Plan ที่ใน Development Plan จะเป็นการแนะนำเนื้อหาเพื่อให้หัวหน้างานเลือกให้กับลูกน้อง แต่ใน Feature : Suggest Content by Competency นี้ จะเป็นการแนะนำเนื้อหาให้กับผู้ใช้โดยตรง และจะเป็นเพียงการเลือกดูข้อมูล ไม่สามารถเลือกให้เข้าไปอยู่ใน Development Plan ผ่านส่วนนี้ได้




User (ฝั่งผู้ใช้งาน)


Scope ID
	Requirement
	Note
	User
	US001
	เมื่อผู้ใช้เข้าสู่เมนู Dashboard เลือกเข้าใช้งานแถบเมนู Growing และเข้าสู่การใช้งานแถบย่อย All Competencies ระบบจะแสดงปุ่ม Suggestion เพิ่มเติมขึ้นมา
	

	US002
	หากผู้ใช้กดเข้าปุ่ม Suggestion แล้ว ระบบจะแสดงหน้ารายการของเนื้อหาที่เกี่ยวข้องกับข้อมูล Competency ซึ่งมีระดับความเชี่ยวชาญเป้าหมายตรงกับ Competency Gap ของผู้ใช้คนนั้นๆ
	

	US003
	ระบบจะมีการแนะนำเนื้อหาอ้างอิงตาม Competency Gap ดังนี้
1. กรณีพนักงานมี competency gap ที่มีข้อมูลทั้ง current proficiency level และ target proficiency level: เนื้อหาที่แนะนำจะ มี 2 แบบ คือ 
- มี proficiency level และ target proficiency level ตรงกับข้อมูลพนักงานคนนั้น
- มี proficiency level เป็น any level และ target proficiency level ตรงกับข้อมูลพนักงานคนนั้น
* ตัวอย่างเช่น user มี competency gap ที่ current proficiency level = 1 และ target proficiency level = 2 
➝ เนื้อหาที่แนะนำจะมี proficiency level = 1/ target proficiency level = 2 และ 
➝ เนื้อหาที่แนะนำจะมี proficiency level = any level/ target proficiency level = 2 ด้วย
   2. กรณีพนักงานคนนั้นๆ มี competency gap ที่ยังไม่มี proficiency level (no assessment result) แต่มี target proficiency level:เนื้อหาที่แนะนำจะมี proficiency level เป็น any level เท่านั้นและ target proficiency level ตรงกับที่ผูก user competency gap
   * ตัวอย่างเช่น user มี competency gap  current proficiency level = no assessment result / target proficiency level = 2 ➝ เนื้อหาที่แนะนำจะมี proficiency level = any level และ target proficiency level = 2
ระบบจะแสดงเนื้อหาทั้งหมดที่ตรงกับเงื่อนไขข้อใดข้อหนึ่ง โดยจะเรียงตาม Last Update ของเนื้อหา (ตามข้อมูลจากระบบ Learn) เนื้อหาที่มีวันเวลาที่อัปเดตล่าสุดจะอยู่ด้านบนสุด
	อ้างอิงจาก UM020.3
https://docs.google.com/document/d/1XzXF73kkpjHDBdukbre2J-lc7r8s7snY/edit?usp=sharing&ouid=110092286145457809119&rtpof=true&sd=true


ที่ต่างกัน คือ เป็นข้อมูลที่แสดงที่ฝั่ง User ตรงๆ ไม่ใช่แสดงที่ Manager เหมือนของ Development Plan
	US004
	เนื้อหาที่นำมาแสดงจะยังไม่มีการกรองสิทธิ์ตาม User Permission และ Publish Status โดยจะแสดงรายการเนื้อหาทั้งหมดรวมถึงเนื้อหาที่อยู่ระหว่างการขอสร้าง (On Requested Content) 
	อ้างอิงจาก US008 https://docs.google.com/document/d/1XzXF73kkpjHDBdukbre2J-lc7r8s7snY/edit?usp=sharing&ouid=110092286145457809119&rtpof=true&sd=true
	US005
	กรณีไม่มีเนื้อหาที่ตรงกับเงื่อนไขในการแนะนำ จะยังแสดงปุ่ม Suggestion ให้ผู้ใช้กดเข้ามาได้ แต่จะแสดงหน้าแบบไม่มีข้อมูล
	

	US006.1
	ข้อมูลที่จะถูกนำมาแสดงในหน้า Suggestion ประกอบด้วย
   * ชื่อ Competency
   * ชื่อหน้า “Content suggestion” หรือเนื้อหาที่แนะนำ
   * รายละเอียดของเนื้อหา ได้แก่ รูปปก (Cover), ชื่อและรหัสของเนื้อหา, ปุ่ม View สำหรับการกดเข้าดูรายละเอียดของเนื้อหา (Content Detail) 
	

	US006.2
	เมื่อกดปุ่ม View จะแสดงรายละเอียดของเนื้อหา (Content Detail) ในแถบหน้าต่างใหม่
	

	US007
	หากผู้ใช้เลือกดูเนื้อหาที่ไม่มีสิทธิ์ตาม User Permission จะแสดง Error : 403 No Permission หรือหากเนื้อหายังไม่ถูกเผยแพร่ (Publish) จะแสดง Error : 404 Page not found
	

	US008
	ประเภทเนื้อหาที่รองรับ มีดังนี้
   * Learning Material 6 ประเภท โดยจะแสดงชื่อดังนี้ 
   * Video, Audio, External Video, Document, Article, Flash Card
   * Course
   * Class: โดยจะมี competency gap ตามกับ Class Program ที่เกี่ยวข้อง
   * Live
   * Test
   * Activity
ไม่มี content type: pathway, Survey, Content type ต่างจาก ConicleX, Public Learning, Public Request และ External Training


โดยจะมีเทียบเท่ากับการแนะนำเนื้อหาบนระบบ Development Plan
	

	US009
	แสดง Class program name ที่รายการเนื้อหาแนะนำที่ content type คือ Class
